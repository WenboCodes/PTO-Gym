# c/ls(2)/ls(4)/d(2)/d(4) 五 layout 图示与互转指令

本文用 ASCII 图示画出 `c()` / `ls(2)` / `ls(4)` / `d(2)` / `d(4)` 五种 layout
的物理形态,并标出**两两之间用什么 micro 指令互转**。配套与依据:

---

## 1. 五种 layout 各是什么

五种 layout 都在回答同一个问题:**256 字节的物理寄存器里,逻辑 lane 怎么排**。
符号沿用 [vmi-layout-tables-illustrated.md](vmi-layout-tables-illustrated.md) §0:
`x0 x1 x2...` 是逻辑 lane 的顺序编号,`__` 是物理寄存器里的空位(padding,
无语义值),`P0/P1/P2/P3` 是物理 part(各 256B)。

| layout | 名字 | 物理含义 | 谁占哪一位 |
|---|---|---|---|
| `c()` | **contiguous**(顺序) | 逻辑 lane 按顺序紧挨着填满,**无空位** | lane `i` 在物理位 `i` |
| `ls(2)` | **lane-stride 2**(间隔留空) | 逻辑 lane 只占**偶**物理位,奇位留空(半满),空位可塞另一路 | lane `i` 在物理位 `2i` |
| `ls(4)` | **lane-stride 4**(每 4 位留 1) | 逻辑 lane 每 4 个物理位只占 1 个(1/4 满),其余留空给另外 3 路 | lane `i` 在物理位 `4i` |
| `d(2)` | **deinterleaved 2**(两路拆 part) | 偶/奇两路**拆到不同 part**,每 part 装一路、各自半满 | 偶 lane 一 part、奇 lane 一 part |
| `d(4)` | **deinterleaved 4**(四路拆 part) | mod4 的四路**拆到四个 part**,每 part 装一路、更稀疏 | mod4=0/1/2/3 各占一 part |

直观对照(同一组逻辑 lane `x0..x127`,16-bit,先看几何):

```
c()  : x0 x1 x2 x3 x4 x5 x6 x7 ...   ← 顺序,无空位,占满
ls(2): x0 __ x2 __ x4 __ x6 __ ...   ← 偶位有效,奇位空(半满,留空给第二路)
ls(4): x0 __ __ __ x4 __ __ __ ...   ← 每 4 位留 1(1/4 满,仅 8-bit)(留空给另 3 路)
d(2) : P0:[x0 x2 x4 ...] P1:[x1 x3 x5 ...]   ← 偶/奇拆到两个 part
d(4) : P0:[x0 x4...] P1:[x1 x5...] P2:[x2 x6...] P3:[x3 x7...]   ← mod4 拆四 part
```

三个要点:

1. **`ls(k)` 与 `d(k)` 是同类几何的两种"住法"**:两者都把一路逻辑 lane
   摊稀(每隔 k 位取 1,或按 mod-k 分路)。差别在**住几个 part**:
   - `ls(k)` 全挤在**一个 part**内,靠"偶位有效/奇位空"留出空位给别路挤进来;
   - `d(k)` 把 k 路**拆到 k 个 part**,每路独占一个 part(各自稀疏但分开住)。
2. **`ls(k)` 必然欠满**:`ls(2)` 半满、`ls(4)` 1/4 满——它牺牲物理 lane 容量换"同
   part 内塞多路"。`d(k)` 不欠 part 数,但每个 part 只装 `1/k` 的 lane。
3. **它们不是随意命名**:`ls(k)` 的 `k` 是**物理位间隔**(stride),`d(k)` 的 `k`
   是**路数**(way)。本文后续的互转表、位宽闸门,都围绕"`ls` 留空"与"`d` 拆 part"
   这两条物理实现展开。

## 2. 先记住一个硬约束

**五种 layout 不可能在同一元素位宽下同时全部物化。** 物化闸门按位宽 + 逻辑
长度 N 限制(表 1):

| layout | 合法位宽 | 需要的 N | 物化路径 |
|---|---|---|---|
| `c()` | 8/16/32/64 | 任意 | 天然 |
| `d(2,1)` | 16/32/64 | 16-bit:256;32/64:128/256 | `vintlv`/`vdintlv`(2 路) |
| `d(4,1)` | **32/64** | 32:256/512;64:128/256/512 | `vintlv`/`vdintlv`(两轮,4 路) |
| `d(2,1)↔d(4,1)` | 32/64 | 32:256/512;64:128/256 | 同上再拆/合 |
| `ls(2)` | **8/16** | 8:1/2/4/8/64/128;16:1/2/4/8/64 | `vbitcast`→`vpack`/`vzunpack`→`vbitcast` |
| `ls(4)` | **仅 8** | 8:1/2/4/8/64 | 同上两轮 |

依据:表 1 [:281-298](../../lib/PTO/Transforms/VMILayoutSupport.cpp#L281-L298);
位宽闸门 [VMIToVPTO.cpp:3289-3290](../../lib/PTO/Transforms/VMIToVPTO.cpp#L3289-L3290)
(`ls(2)` 要 bits∈{8,16}、`ls(4)` 要 bits==8)。

→ 下文分两套示例,**没有一个能画全五种**:
- **A. 32-bit, N=256(2 part)** — 画出 `c / d(2) / d(4)` 三种(d 系 32-bit 全在此)。
- **B. 8-bit, N=128(1 part)** — 画出 `c / ls(2) / ls(4)` 三种(ls 系全在此)。
- 两套之间靠**跨位宽 cast**(§6)连 `ls` 与 `d`。

> 物理:一个 part `P` = 256B = 8 个 `BL`(VCG block,各 32B)。
> 32-bit:64 lane/P,8 lane/BL;8-bit:256 lane/P,32 lane/BL。

## 3. 五 layout 物理图示

### 3.1 示例 A — 32-bit,逻辑 256 lane(2 个 part):c / d(2) / d(4)

```
c() — contiguous,顺序填满 2 part:
 P0: [y0  y1  y2 ... y63]            ← 64 lane,8 lane/BL × 8 BL
 P1: [y64 y65 y66 ... y127]          ← (此处 N=256 用 y128..y255 跨 4 part,
                                      为了画 d(2)/d(4) 下面用 mod 标号)

为画清交错,用"逻辑 lane 在哪个物理位"标号(mod):
 c() N=256 跨 4 part(每 part 64 lane):
  P0:[y0..y63]  P1:[y64..y127]  P2:[y128..y191]  P3:[y192..y255]

d(2,1) — 2 路解交错,偶/奇拆到不同 part(每 part 64 lane,半满):
  P0(偶): [y0  y2  y4 ... y126]      ← 全偶
  P1(奇): [y1  y3  y5 ... y127]      ← 全奇
  P2(偶): [y128 y130 ... y254]
  P3(奇): [y129 y131 ... y255]

d(4,1) — 4 路解交错,mod4 拆到四个 part(每 part 32 lane,更稀疏):
  P0(mod4=0): [y0  y4  y8 ... y252]
  P1(mod4=1): [y1  y5  y9 ... y253]
  P2(mod4=2): [y2  y6 y10 ... y254]
  P3(mod4=3): [y3  y7 y11 ... y255]
```

### 3.2 示例 B — 8-bit,逻辑 128 lane(1 part):c / ls(2) / ls(4)

```
c() — contiguous,顺序填满:
┌────────────────────────────────────────────────────────┐
│ x0 x1 x2 ... x31 │ x32 ... x63 │ ... │ x96 ... x127 │   (只画前 1 part 内 128 lane)
└────────────────────────────────────────────────────────┘
     BL0(32 lane)   BL1        ...        BL3
                         P0 (256B, 128 lane 全满)

ls(2) — lane-stride 2,只占偶物理位(半满,奇位留空):
┌──────────────────────────────────────────────────────┐
│ x0 __ x2 __ x4 __ ... x62 __ │ (奇位空,可塞第二路)      │
└──────────────────────────────────────────────────────┘
        64 个有效 lane 占偶位
                         P0 (半满)

ls(4) — lane-stride 4,每 4 位留 1(仅 8-bit):
┌────────────────────────────────────────────────────────┐
│ x0 __ __ __ x4 __ __ __ x8 __ __ __ ... x60 __ __ __ │
└────────────────────────────────────────────────────────┘
        每 4 物理位放 1 个有效 lane,32 个有效 lane
                         P0 (1/4 满)
```

## 4. 寄存器内互转指令表(同位宽,表 1 物化)

同元素位宽下,layout 互转靠**寄存器内重排**(`vintlv`/`vdintlv`/`vpack`/`vzunpack`,
不访存)。下表只列**表 1 合法**的成对转换(✓),其余标 ✗(同位宽不物化)。

| 从 → 到 | `c()` | `ls(2)` | `ls(4)` | `d(2)` | `d(4)` |
|---|---|---|---|---|---|
| **`c()`** | — | ✓ 8/16-bit | ✓ 8-bit | ✓ 16/32/64-bit | ✓ 32/64-bit |
| **`ls(2)`** | ✓ 8/16-bit | — | ✗ | ✗ | ✗ |
| **`ls(4)`** | ✓ 8-bit | ✗ | — | ✗ | ✗ |
| **`d(2)`** | ✓ 16/32/64-bit | ✗ | ✗ | — | ✓ 32/64-bit |
| **`d(4)`** | ✓ 32/64-bit | ✗ | ✗ | ✓ 32/64-bit | — |

> **关键**:`ls(k)` 与 `d(*)` 之间在**同位宽**下**没有寄存器内直转**(表 1 全为 ✗)。
> `ls↔c`、`d↔c`、`d(2)↔d(4)` 各自可物化;要连 `ls` 与 `d`,走**跨位宽 cast**(§6)或
> **访存 dist**(§7)。

### 4.1 各 ✓ 转换的指令链路

| 转换 | 位宽 | micro 指令链路 | 依据 |
|---|---|---|---|
| `c() ↔ d(2,1)` | 16/32/64 | `pto.vdintlv` / `pto.vintlv`(2 路) | 表 1 :281-286;micro-12 |
| `c() ↔ d(4,1)` | 32/64 | `pto.vdintlv` / `pto.vintlv`(两轮,4 路) | 表 1 :287-290;micro-12 |
| `d(2,1) ↔ d(4,1)` | 32/64 | `pto.vdintlv` / `pto.vintlv`(再拆/再合) | 表 1 :295-298 |
| `c() ↔ ls(2)` | **8/16** | `vbitcast`(fp→uint)→`vzunpack`/`vpack`→`vbitcast`(回 fp) | 表 1 :299-302 |
| `c() ↔ ls(4)` | **仅 8** | 同上两轮 | 表 1 :303-304 |

> fp16/f8 走 `vpack`/`vzunpack` 靠 `vbitcast` 借路成 uint:lane 重排只动位置不改值,
> fp 与 uint 结果一致。**限制按位宽不按类型**:`ls(2)` 要 bits∈{8,16}、`ls(4)` 要
> bits==8([VMIToVPTO.cpp:3289-3290](../../lib/PTO/Transforms/VMIToVPTO.cpp#L3289-L3290))。

## 5. 寄存器内互转图示

### 5.1 `c() ↔ d(2,1)` — 两路交错/解交错(示例 A,32-bit)

```
c() (256 lane, 4 part):                 d(2,1) (偶/奇拆 4 part):
 P0:[y0..y63]                            P0(偶):[y0 y2 ... y126]
 P1:[y64..y127]      ──vdintlv──▶        P1(奇):[y1 y3 ... y127]
 P2:[y128..y191]                         P2(偶):[y128 y130 ...]
 P3:[y192..y255]                         P3(奇):[y129 y131 ...]
                ◀──vintlv───   (两路合回 contiguous)
```
**指令**:`pto.vdintlv %lhs,%rhs → %even,%high`(解交错)、`pto.vintlv %a,%b → %low,%high`(交错)。micro-12。

### 5.2 `c() ↔ ls(2)` — 偶位 packing(示例 B,8-bit)

```
c() (64 lane 有效, f8):                 ls(2) (同一 part,偶位):
┌──────────────────────────┐             ┌──────────────────────────────────┐
│ x0 x1 x2 x3 ... x62 x63 │  ──pack──▶   │ x0 __ x2 __ x4 __ ... x62 __    │
└──────────────────────────┘             └──────────────────────────────────┘
        P0(64 lane 顺序)                          P0(半满,奇位空)
                              ◀──zunpack──
```
**指令链路**(fp8):`vbitcast`(f8→ui8)→ `vpack`/`vzunpack`(lane 位排列)→ `vbitcast`(ui8→f8)。
micro-12。> 不能从 `ls(2)` 直接到 `d(2)`:表 1 `ls(2)↔d(2)` 同位宽为 ✗。要连两者,必经
> `ls(2)→c()→d(2)` 两步,或走跨位宽 cast(§6)。

### 5.3 `c() ↔ ls(4)` — 每 4 位留 1(示例 B,8-bit,两轮 packing)

```
c() (32 lane 有效):     ls(4):
┌──────────────┐       ┌──────────────────────────────────┐
│ x0 x1 ... x31│ ──pack(两轮)──▶ │ x0 __ __ __ x4 __ __ __ ... x28 __ __ __│
└──────────────┘       └──────────────────────────────────┘
   P0(顺序)                    P0(1/4 满,每 4 位留 1)
                       ◀──zunpack(两轮)──
```

### 5.4 `d(2,1) ↔ d(4,1)` — 两路再拆四路(示例 A,32-bit)

```
d(2,1):                          d(4,1):
 P0=偶 [y0 y2 y4 ...]    ──再拆─▶ P0=mod4=0 [y0 y4 y8 ...]
 P1=奇 [y1 y3 y5 ...]            P1=mod4=1 [y1 y5 y9 ...]
 P2=偶 [y128 ...]                P2=mod4=2 [y2 y6 y10..]
 P3=奇 [y129 ...]                P3=mod4=3 [y3 y7 y11..]
                    ◀──合(两轮 vintlv)──
```
**指令**:`vdintlv`/`vintlv`(在已有 d(2) 的 part 上再拆/合)。micro-12。

## 6. ls ↔ d 的真正通路:跨位宽 cast(表 4)

`ls(k)` 与 `d(*)` 同位宽无直转,但**在元素位宽变化(cast)时可以配对**。下表是
`kLegalCastLayoutPatterns`(:409)里 `ls`/`d` 交叉的合法 cast 对:

| cast 方向 | 源 → 结果 layout | micro | 依据 |
|---|---|---|---|
| 16→32 宽化 | `ls(2) → c()` | `vcvt{EVEN}` 一次 | 表 4 :413 |
| 16→32 宽化 | `c() → d(2)` | `vcvt{EVEN}+{ODD}` | 表 4 :412 |
| 8→32 宽化 | `ls(2) → d(2)` | `vcvt` 系列 | 表 4 :422 |
| 8→32 宽化 | `ls(4) → c()` | `vcvt` | 表 4 :423 |
| 32→16 窄化 | `c() → ls(2)` | `vcvt{part}` | 表 4 :418 |
| 32→16 窄化 | `d(2) → c()` | `vcvt{part}` | 表 4 :417 |
| 32→8 窄化 | `c() → ls(4)` | `vcvt` | 表 4 :421 |
| 32→8 窄化 | `d(2) → ls(2)` | `vcvt` | 表 4 :424 |

> 例:`ls(2)@f16` 想到 `d(2)@f32` 的等价 layout,直接走一次 `vcvt` 宽化(表 4
> `ls(2)→d(2)` @8→32),而不是先寄存器内 `ls(2)→c()` 再 `c()→d(2)` 两步。**cast 的
> part 机制同时改了位宽和 layout**(见 [vmi-layout-tables-illustrated.md](vmi-layout-tables-illustrated.md)
> 表 3 精确行 vs 默认行)。这就是示例 A(32-bit d 系)与示例 B(8/16-bit ls 系)之间的桥。

## 7. 访存侧:load/store 直接产出/消费 ls/d

除了寄存器内互转和 cast,load/store 自己的 dist token 也能**直接**产出或消费这些 layout,
绕开寄存器内重排([VMIToVPTO.cpp:3078-3104](../../lib/PTO/Transforms/VMIToVPTO.cpp#L3078-L3104)):

| layout | load dist(产出) | store dist(消费) | 元素限制 |
|---|---|---|---|
| `c()` | `NORM` | `NORM_B*` | 8/16/32 |
| `ls(2)` | `UNPK_B*` | `PK_B16/B32/B64` | 8/16/32(store 按位宽选 PK 名) |
| `ls(4)` | `UNPK4` | `PK4_B32` | **仅 8** |
| `d(2)`/`d(2,1)` | `DINTLV_B*`(`pto.vldsx2`) | `INTLV_B*`(`pto.vstsx2`) | 8/16/32 |
| `d(4)`/`d(4,1)` | `BDINTLV`/多轮 | `INTLV`/多轮 | 8/16/32 |

> `ls(4)` 的 store 是 **`PK4_B32`**(不是 `PK_B*`):32-bit 元素取低 8 bit = 每 4 lane 留 1。
> `ls(2)` store 按元素位宽分别叫 `PK_B16`(8-bit)/`PK_B32`(16-bit)/`PK_B64`(32-bit)。
> 这些 dist token 是**另一条**连 ls 与 d 的通路:load 直接产 `d(2)`,store 直接消 `ls(2)`,
> 全程不必在寄存器内互转。

## 8. 五 layout 全景图

```
  ┌───────────────────────────────────────────────────────────┐
  │ 跨位宽 cast(表 4)连 ls 系与 d 系;访存 dist(§7)也能直连       │
  │   ls(2) ──vcvt(8→32)──▶ d(2)        ls(4) ──vcvt(8→32)──▶ c()
  │   d(2)  ──vcvt(32→8)──▶ ls(2)       c()  ──vcvt(32→8)──▶ ls(4)
  └───────────────────────────────────────────────────────────┘

  寄存器内(表 1, 同位宽):

     ┌──[8-bit, 1 part]────────────────────────────────┐
     │   c() ◀──vpack/vzunpack(经 vbitcast)──▶ ls(2)   │
     │   c() ◀──(两轮)──────────────────────▶ ls(4)   │
     │   (ls(2)↔ls(4) 无直转,必经 c())                │
     └────────────────────────────────────────────────┘
            │ 跨位宽 cast(8→32) 把 ls 系接到 d 系 ↓
     ┌──[32-bit, 2-4 part]────────────────────────────┐
     │   c() ◀──vintlv/vdintlv──▶ d(2) ◀──再拆/合──▶ d(4) │
     └────────────────────────────────────────────────┘

  访存侧(dist, 直产/直消, 绕开寄存器内):
   load:  NORM→c, UNPK_B*→ls(2), UNPK4→ls(4), DINTLV_B*→d(2), BDINTLV→d(4)
   store: NORM_B*←c, PK_B*←ls(2), PK4_B32←ls(4), INTLV_B*←d(2), INTLV←d(4)
```

## 9. 一句话总结

- **`c / d(2) / d(4)`** 是"多路交错"一族,寄存器内靠 `vintlv`/`vdintlv` 互转
  (16/32/64-bit;`d(4)` 与 `d(2)↔d(4)` 需 32/64-bit)。
- **`c / ls(2) / ls(4)`** 是"间隔留空"一族,寄存器内靠 `vpack`/`vzunpack`(经
  `vbitcast` 借路)互转,`ls(2)` 需 8/16-bit、`ls(4)` 仅 8-bit。
- **`ls(k)` 与 `d(*)` 同位宽不能直转**;要连它们,走**跨位宽 cast**(表 4)或
  **访存 dist token**(load/store 直接产出/消费)。
- 五种 layout 不可能在单一元素位宽下同时物化:32-bit 画 `c/d(2)/d(4)`(d 系),
  8/16-bit 画 `c/ls(2)/ls(4)`(ls 系),两者由跨位宽 cast 桥接。
